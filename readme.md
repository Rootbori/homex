ออกแบบ Web Application แบบ Mobile-first สำหรับระบบหาช่างแอร์ + CRM จัดการงานร้านช่างแอร์ในประเทศไทย

เป้าหมายของระบบ:
เป็นระบบที่มี 2 ฝั่งในระบบเดียว

1. ฝั่งลูกค้า

- ค้นหาช่างแอร์ได้
- ดูรายชื่อช่างทั้งหมด
- ดูโปรไฟล์ช่าง
- ดูบริการ รีวิว พื้นที่ให้บริการ
- ส่งคำขอใช้บริการ
- นัดหมายและติดตามสถานะงาน

2. ฝั่งช่าง / ร้าน

- รับลูกค้าจาก LINE Official Account และจากหน้าค้นหาช่าง
- จัดการ lead, job, quotation, schedule
- ช่างแต่ละคนเห็นเฉพาะลูกค้าและงานของตัวเอง
- ไม่เห็นลูกค้าของช่างคนอื่น

ระบบนี้ไม่ใช่ marketplace แบบซับซ้อน
แต่เป็นระบบใช้งานง่ายสำหรับ startup MVP
เน้นการใช้งานผ่านมือถือเป็นหลัก

กลุ่มผู้ใช้:

- ลูกค้าที่ต้องการหาช่างแอร์
- เจ้าของร้านช่างแอร์
- แอดมินร้าน
- dispatcher
- ช่าง

Flow หลักของระบบ:
ลูกค้าค้นหาช่าง → ดูโปรไฟล์ช่าง → ส่งคำขอ → ร้าน/ช่างรับ lead → เก็บข้อมูล → ออกใบเสนอราคา → ลูกค้าตกลง → นัดหมาย → มอบหมายช่าง → ทำงาน → ปิดงาน

ข้อกำหนดสำคัญ:

- ต้องเป็น mobile-first responsive web app
- ใช้งานผ่านมือถือได้ดี
- ใช้งานง่าย
- ปุ่มใหญ่
- อ่านง่าย
- กดไม่กี่ครั้ง
- ลดความซับซ้อน
- รองรับผู้ใช้ที่ไม่เก่งเทคโนโลยี

แนวทางการออกแบบ:

- UI แบบ modern SaaS + service platform
- เรียบ สะอาด มืออาชีพ
- ใช้โทนสีอ่อน ขาว ฟ้า
- เน้น card-based layout
- ใช้ bottom navigation
- ใช้ sticky action buttons
- ใช้ chips, cards, stacked sections, accordions
- หลีกเลี่ยง wide tables แบบ desktop

แยกประสบการณ์ใช้งานเป็น 2 ฝั่ง

====================
ฝั่งที่ 1: ลูกค้า
====================

หน้าที่ต้องออกแบบสำหรับลูกค้า:

1. หน้า Home / Find Technician

- search bar สำหรับค้นหาช่าง
- filter:
  - พื้นที่
  - ประเภทบริการ
  - ราคาเริ่มต้น
  - rating
  - available now
- list ช่างแบบ card
- แต่ละ card แสดง:
  - รูปช่างหรือโลโก้ร้าน
  - ชื่อช่าง / ชื่อร้าน
  - rating
  - จำนวนรีวิว
  - พื้นที่ให้บริการ
  - บริการที่รับ
  - ราคาเริ่มต้น
  - ป้าย available / busy
- ปุ่ม:
  - ดูโปรไฟล์
  - ส่งคำขอใช้บริการ

2. หน้า Technician Profile

- รูปช่าง / โลโก้
- ชื่อ
- rating
- รีวิว
- ประสบการณ์
- พื้นที่ให้บริการ
- รายการบริการ:
  - ล้างแอร์
  - ซ่อมแอร์
  - เติมน้ำยา
  - ติดตั้ง
- ราคาเริ่มต้น
- เวลาทำการ
- ผลงาน / รูปงานที่ผ่านมา
- ปุ่ม:
  - ส่งคำขอ
  - โทร
  - แชทผ่าน LINE

3. หน้า Create Service Request

- ฟอร์มสั้น ใช้งานง่าย
- ข้อมูลที่ต้องกรอก:
  - ชื่อ
  - เบอร์โทร
  - พื้นที่
  - ที่อยู่
  - ประเภทบริการ
  - จำนวนเครื่อง
  - BTU
  - ยี่ห้อ
  - อาการ
  - รูป
  - วันที่สะดวก
  - เวลาที่สะดวก
  - หมายเหตุ
- ใช้ dropdown / chips / quick select เพื่อลดการพิมพ์
- ปุ่มส่งคำขอเด่นชัด

4. หน้า My Requests / My Jobs

- list งานของลูกค้า
- แสดงสถานะ:
  - รอร้านรับงาน
  - รอใบเสนอราคา
  - รอยืนยัน
  - นัดหมายแล้ว
  - กำลังไป
  - กำลังทำ
  - เสร็จแล้ว
  - ยกเลิก
- แต่ละ card แสดง:
  - ช่าง / ร้าน
  - ประเภทงาน
  - วันนัด
  - สถานะ
  - ราคารวม
- กดดูรายละเอียดได้

5. หน้า Job Tracking for Customer

- แสดง:
  - ข้อมูลช่าง
  - สถานะงาน
  - timeline
  - ใบเสนอราคา
  - วันนัด
  - ที่อยู่
  - รูปงาน
- ปุ่ม:
  - ยืนยันใบเสนอราคา
  - ติดต่อช่าง
  - ยกเลิกงาน
  - รีวิวหลังจบงาน

====================
ฝั่งที่ 2: ร้าน / แอดมิน / ช่าง
====================

แยก role ชัดเจน:

- Admin / Owner
- Dispatcher
- Technician

สิทธิ์การมองเห็น:

- Admin เห็นทุก lead ทุก job ทุกลูกค้าในร้าน
- Dispatcher เห็นทุกงานในร้านและสามารถ assign ได้
- Technician เห็นเฉพาะงาน ลูกค้า และข้อมูลที่ถูก assign ให้ตัวเองเท่านั้น
- Technician ต้องไม่เห็นลูกค้าหรือ lead ของช่างคนอื่น

หน้าที่ต้องออกแบบสำหรับฝั่งร้าน/ช่าง:

1. Dashboard

- KPI:
  - lead ใหม่วันนี้
  - งานกำลังทำ
  - ใบเสนอราคาค้าง
  - งานเสร็จแล้ว
  - รายได้เดือนนี้
- งานวันนี้
- งานด่วน
- lead ล่าสุดจาก LINE และจากหน้าหาช่าง
- ใช้ card ซ้อนแนวตั้ง อ่านง่ายบนมือถือ

2. Leads Page

- list lead แบบ card
- แต่ละ lead card แสดง:
  - ชื่อลูกค้า
  - เบอร์
  - พื้นที่
  - ประเภทบริการ
  - จำนวนเครื่อง
  - อาการ
  - source:
    - LINE OA
    - Find Technician
  - เวลา
  - สถานะ
- filter:
  - วันที่
  - สถานะ
  - พื้นที่
  - source
- detail lead:
  - ข้อมูลลูกค้า
  - ข้อมูลแอร์
  - อาการ
  - รูป
  - เวลาที่สะดวก
  - ปุ่ม:
    - สร้างใบเสนอราคา
    - assign ช่าง
    - เปลี่ยนเป็น job

3. Job Detail Page

- header:
  - job ID
  - สถานะ
  - ชื่อลูกค้า
- section:
  - ข้อมูลลูกค้า
  - ข้อมูลแอร์
  - อาการ
  - ที่อยู่ + แผนที่
  - รูป
  - ใบเสนอราคา
  - ช่างที่รับงาน
  - note ภายใน
  - payment status
- timeline:
  - lead created
  - quoted
  - confirmed
  - scheduled
  - on the way
  - in progress
  - completed
- sticky bottom actions:
  - กำลังไป
  - เริ่มงาน
  - ปิดงาน

4. Quotation Page

- หน้าออกใบเสนอราคาแบบมือถือ
- ปุ่มเลือกบริการสำเร็จรูป:
  - ล้างแอร์ 9000-12000 BTU
  - ล้างแอร์ 12000-24000 BTU
  - เติมน้ำยา
  - ซ่อม
  - ติดตั้ง
- เพิ่มรายการเองได้
- สรุปราคา
- note
- ปุ่ม:
  - ส่งผ่าน LINE
- แสดงสถานะใบเสนอราคา

5. Schedule Page

- ปฏิทินมือถือ
- ดูรายวัน / รายสัปดาห์
- งานตามเวลา
- แสดงช่างที่รับงาน
- สีแยกตามสถานะ
- เลื่อนนัดได้ง่าย

6. Technician Page

- list ช่างแบบ card
- แสดง:
  - ชื่อ
  - skill
  - rating
  - พื้นที่
  - availability
- technician detail:
  - งานที่ได้รับ
  - performance
  - ประวัติงาน

7. Technician My Jobs

- หน้าสำหรับช่าง
- เห็นเฉพาะงานของตัวเอง
- list งานที่ได้รับมอบหมาย
- แต่ละ card แสดง:
  - ชื่อลูกค้า
  - พื้นที่
  - เวลา
  - ประเภทงาน
  - สถานะ
- ปุ่ม:
  - ดูรายละเอียด
  - โทรหาลูกค้า
  - เปิดแผนที่
  - เปลี่ยนสถานะงาน
  - อัปโหลดรูปก่อน/หลัง

8. Customers Page

- สำหรับ admin เท่านั้น
- list ลูกค้า
- profile ลูกค้า:
  - ข้อมูลติดต่อ
  - ที่อยู่
  - ประวัติแอร์
  - งานที่ผ่านมา
  - ยอดใช้จ่าย
  - note

Feature สำคัญ:

- รองรับ lead จาก LINE OA
- รองรับ lead จากหน้าค้นหาช่าง
- แสดง source ของ lead ชัดเจน
- ส่งใบเสนอราคาผ่าน LINE ได้
- แสดง preview ข้อความจาก LINE
- ใช้ field เฉพาะแอร์:
  - BTU
  - brand
  - symptom
  - จำนวนเครื่อง

Navigation:
สำหรับลูกค้า:

- Home
- Search
- My Requests
- Profile

สำหรับร้าน/ช่าง:

- Dashboard
- Leads
- Jobs
- Schedule
- More

ข้อสำคัญ:

- ต้องออกแบบให้ mobile-first จริง
- ใช้งานง่ายมาก
- ลดความซับซ้อน
- ปุ่มใหญ่

- ไม่ซับซ้อนเกิน
- เน้นใช้งานจริง
- แยก role และสิทธิ์การมองเห็นชัดเจน
